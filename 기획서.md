# 대화형 AI 완전판매 검증 에이전트 — 기획서

> 우리은행 해피콜을 "버튼 선택"에서 "AI 음성 대화"로 진화시켜 불완전판매 리스크를 원천 차단합니다.

---

## 1. 문제 정의 (Pain Point)

### 현행 해피콜의 구조적 한계

| 항목 | 현행 방식 | 문제점 |
|------|-----------|--------|
| 인터페이스 | 8개 질문 버튼 클릭 | 읽지 않고 기계적 클릭 가능 |
| 이해 확인 | 없음 | "예" 클릭이 진정한 이해의 증거가 되지 않음 |
| 분쟁 발생 시 | 클릭 로그만 존재 | "설명을 들었지만 이해 못했다" 주장 반박 불가 |
| 규제 대응 | 형식적 서명 | 금융소비자보호법상 "충분한 설명 의무" 실질 이행 어려움 |

### 핵심 Pain Point

**금융사 입장:** 불완전판매 분쟁 시 "고객이 진정으로 이해했다"는 증거가 없어 배상 책임.  
**고객 입장:** 복잡한 위험 내용을 읽지 않고 지나쳐 손실 발생 후 뒤늦은 후회.  
**규제 당국 입장:** 형식적 동의 절차만으로는 금융소비자보호법 취지 달성 불가.

---

## 2. 솔루션 개요

### AI 완전판매 검증 에이전트

> **"고객이 실제로 이해했는지"를 AI가 음성으로 확인하고, LLM이 진정성을 판별합니다.**

```
[기존 흐름]
 가입 신청 → 버튼 클릭 해피콜 → 완료

[신규 흐름]
 가입 신청
   → RAG: 고객 맞춤 위험 브리핑 생성
   → TTS: AI가 위험 내용 음성 안내
   → STT: 고객 음성 녹음
   → LLM Judge: 진정한 이해·동의 판별
   → PASS: 음성 동의 기록 + 완전판매 완료
   → FAIL: AI 재설명 (최대 3회) → 히든 디버그 텍스트 입력
```

## 3. 핵심 기능

### 3-1. RAG 기반 핵심 위험 추출

투자자 유형과 펀드 위험등급을 매칭하여 **개인화된 위험 경고**를 생성합니다.

- `fund_risks.json`에 5개 펀드(9개 클래스) 핵심 위험 데이터 사전 구축
- 투자자 성향(안정형~공격투자형) 5단계 × 펀드 위험등급(낮음~매우높음) 5단계 매핑
- 불일치 심각도에 따라 3단계 경고 생성:
  - **일치:** "적합한 수준입니다. 단, 원금 손실 가능성은 항상 존재합니다."
  - **경미 불일치 (1단계):** "[주의] 성향보다 위험등급이 다소 높습니다."
  - **심각 불일치 (2단계 이상):** "[중요 경고] 전문 상담사와 상의를 강력히 권고합니다."
- ChromaDB 벡터스토어 연동 (설치 시 자동 활성화) → 향후 약관 전문 RAG 확장 기반

### 3-2. 양방향 음성 서명 (TTS + STT)

AI가 **말하고** 고객이 **말로 답하는** 완전한 양방향 음성 인터페이스.

**TTS (AI 발화):**
- gTTS 기본 엔진 (Google TTS, 한국어 최적화, 즉시 동작)
- Qwen3-TTS 0.6B 오프라인 고품질 엔진 준비 (RTX 5060 Ti 8GB 환경 지원, 코드 주석 제공)
- Web Speech API fallback (서버 오프라인 시)

**STT (고객 발화 인식):**
- MediaRecorder API로 webm 녹음 → FastAPI 전송
- faster-whisper (large-v3 / base) — 한국어 특화, VAD 필터
- GPU 우선(float16) → CPU fallback(int8) 자동 전환
- Web Speech API fallback (서버 미연결 시)

**음성 동의 기록:**
- 타임스탬프 포함 오디오 로그 → localStorage 저장
- 인식 텍스트 + Judge 결과 포함 감사 로그

### 3-3. LLM Judge 기반 완전판매 검증

**"네"라고 말한다고 PASS가 아닙니다.** 진정한 이해와 동의인지를 LLM이 판별합니다.

| 발화 유형 | 판정 | 근거 |
|-----------|------|------|
| "네, 이해했습니다" | PASS | 명확한 긍정 |
| "네" (단답) | PASS | 명확한 긍정 |
| "아마도요" | FAIL | 불확실한 표현 |
| "잘 모르겠는데요" | FAIL | 이해 미달 |
| "그러니까요?" | FAIL | 재확인 요청 |
| "네, 근데 혹시..." | FAIL | 조건부 동의 |
| (침묵/소음) | FAIL | 유효 발화 없음 |

**Gemini 2.0 Flash** 사용 (비용 효율 최적):
- GEMINI_API_KEY 설정 시 LLM 판별
- 미설정 시 로컬 정규식 휴리스틱 자동 fallback
- 응답 형식: `{ result, reason, suggestion }` 구조화 JSON

## 6. 평가 기준별 전략

### 6-1. 효과성 40점: 불완전판매 방어 증거

**기존 방식의 한계와 우리의 해답:**

```
[기존] 클릭 로그만 있음
  → "제가 눌렀지만 내용을 이해한 게 아닙니다" → 분쟁 발생

[신규] 음성 발화 + LLM 판별 결과 + 타임스탬프 로그
  → "고객님이 '네, 이해했습니다'라고 말씀하셨고,
     AI가 진정한 동의로 판별했습니다" → 분쟁 방어
```

**효과 지표:**
- 불완전판매 분쟁 시 음성 기반 동의 근거 제시 가능
- 투자자 성향 불일치 펀드 가입 시 강화된 경고 자동 삽입
- 조건부 동의("네, 근데...") FAIL 처리로 진정한 이해 미달 차단
- 3회 FAIL 시 텍스트 입력으로 전환하여 완전판매 강제 확인

### 6-2. 실현가능성 30점: 스트레스 테스트 결과

**Graceful Degradation 설계 — 오프라인에서도 동작:**

```
[Level 1] 전체 스택 정상
  Gemini + faster-whisper + gTTS → 최고 품질

[Level 2] Gemini API 없음
  로컬 키워드 휴리스틱 → 기본 PASS/FAIL 판별 가능

[Level 3] 백엔드 서버 오프라인
  Web Speech API (브라우저) + 로컬 키워드 판별 → 데모 지속 가능

[Level 4] 마이크 권한 없음
  Web Speech API fallback → 히든 디버그 텍스트 입력

[Level 5] 모든 음성 불가
  히든 디버그 버튼 3초 롱프레스 → 텍스트로 판별
```

**RTX 5060 Ti 8GB 최적화:**
- Whisper large-v3: float16, ~3GB VRAM, 실시간 처리
- Whisper base: int8, ~150MB VRAM, 100ms 이하 지연 (데모 권장)
- GPU 미감지 시 CPU int8 자동 전환

### 6-3. 혁신성 30점: HRI(Human-Robot Interaction) 관점

**인간-AI 신뢰 설계 원칙 적용:**

| HRI 원칙 | 적용 내용 |
|-----------|-----------|
| 의도 투명성 | AI가 무엇을 묻고 왜 묻는지 명확히 설명 |
| 오류 복구 | FAIL 시 이유 설명 + 재시도 기회 3회 제공 |
| 인간 주도권 | 3회 FAIL 후 텍스트 입력으로 인간이 제어권 회수 |
| 적절한 난이도 | 복잡한 약관을 3줄 요약으로 인지 부하 최소화 |
| 음성 자연성 | gTTS 0.95배속 + 문장 구조 최적화 |
| 시각 피드백 | 음성 파형 애니메이션 + 단계별 색상 변화 |

**금융 AI 신뢰도 향상 요소:**
- 투자자 성향과 펀드 매칭 결과를 시각적으로 표시
- AI 판단 근거(reason)를 고객에게 공개 (블랙박스 아님)
- 불일치 경고 3단계 세분화 → 과도한 공포심 vs. 무시 사이 균형

## 7. 데모 시나리오 (발표 스크립트)

### 전제 설정
- 고객: 이혜원 (위험중립형)
- 선택 펀드: 삼성 글로벌 반도체 (매우높은위험)
- 가입 금액: 100,000원

### 시나리오 A — 정상 완전판매 (2분)

```
1. [Page08 진입]
   "AI 완전판매 검증이 시작됩니다. RAG가 고객 맞춤 위험 정보를 추출합니다."

2. [위험 브리핑 TTS 자동 재생]
   AI: "고객님, 삼성 글로벌 반도체는 매우높은위험 상품입니다.
        위험중립형이신 고객님의 성향과 다소 차이가 있을 수 있습니다.
        원금 전액 손실이 가능하며, 환율 변동 위험도 있습니다.
        이해하셨다면 '네, 이해했습니다'라고 말씀해 주세요."

3. [마이크 버튼 클릭]
   고객: "네, 이해했습니다"

4. [LLM Judge → PASS]
   화면: 초록 체크 + "완전판매가 확인되었습니다"

5. [동의 완료 버튼 → 파란색 활성화]
   → 해피콜 등록 완료
```

### 시나리오 B — 불완전 동의 감지 (3분)

```
1. [마이크 버튼 클릭]
   고객: "아... 잘 모르겠는데요"

2. [LLM Judge → FAIL]
   이유: "불확실한 표현이 감지되었습니다"
   AI 재설명: "원금 손실 가능성을 명확히 이해하셨나요?
               '네, 이해했습니다'라고 말씀해 주시겠어요?"
   진행 표시: 1/3

3. [2차 시도]
   고객: "이해했습니다"
   → PASS → 완전판매 확인
```

### 시나리오 C — 히든 디버그 모드 (1분)

```
1. 3회 모두 음성 인식 실패

2. 화면 우하단 5×5px 투명 버튼을 3초 롱프레스

3. 텍스트 입력창 등장

4. "네, 이해했습니다" 입력 → 제출

5. 동일한 /api/judge 검증 → PASS
```

## 8. 비상 대비 (히든 디버그 모드)

발표 현장에서 네트워크 불안정 또는 음성 인식 실패 시 대비책.

### 활성화 방법
- 화면 우하단 구석 5×5px 투명 버튼을 **3초간 길게 누름** (터치/마우스 모두 지원)
- 또는 3회 연속 FAIL 시 자동으로 텍스트 입력창 등장

### 히든 모드 기능
- 텍스트 입력창 + "텍스트로 제출" 버튼
- 입력 텍스트도 동일하게 `/api/judge` 검증 통과
- "네, 이해했습니다" 입력 → 정상 PASS 처리

### Graceful Degradation 체인
```
백엔드 정상 → faster-whisper → gTTS
     ↓ (서버 오프라인)
Web Speech API + 키워드 휴리스틱
     ↓ (마이크 없음)
히든 디버그 텍스트 입력
```

## 9. 향후 확장 로드맵

### Phase 1 — 현재 (데모 완성)
- ✅ RAG 기반 위험 추출 (fund_risks.json)
- ✅ TTS + STT 음성 서명
- ✅ LLM Judge (Gemini / 휴리스틱 fallback)
- ✅ 히든 디버그 모드

### Phase 2 — 3개월 내 (고도화)
- 약관 전문 PDF → ChromaDB 벡터 인덱싱 (LangChain TextSplitter)
- 고객 발화 "약관에 없는 질문" 감지 → 정확한 FAIL 처리
- 음성 동의 파일 서버 저장 (분쟁 증거 자료)
- 다국어 지원 (영어, 중국어 투자자)

### Phase 3 — 6개월 내 (고도화)
- 실시간 스트리밍 STT (WebSocket 기반 latency 최소화)
- Qwen3-TTS 온프레미스 배포 (인터넷 불필요)
- 판매원 모니터링 대시보드 (이상 패턴 알림)
- 규제 보고서 자동 생성 (금감원 제출용)

### Phase 4 — 1년 내 (플랫폼화)
- 타 금융사 API 제공 (SaaS 모델)
- 대면 판매원 태블릿 통합 (오프라인 채널)
- 멀티모달 확장 — 화상 표정 분석 + 음성 결합 판별
- 규제 기술(RegTech) 인증 취득
